# Contributing

We welcome contributions to the VSSV database engine! Please follow these guidelines to ensure a smooth and effective development process.

## Development Workflow

1.  **Fork the repository** and create a new branch for your feature or bug fix.
2.  **Make your changes** and ensure all new code is covered by tests.
3.  **Follow the coding style** and conventions used throughout the project.
4.  **Write clear and concise commit messages** using the [Conventional Commits](https://www.conventionalcommits.org/) format.
5.  **Submit a pull request** with a detailed description of your changes.

## Code Style

-   **Java:** Follow standard Java coding conventions.
-   **Javadoc:** All public classes, methods, and non-obvious fields should be documented with Javadoc.

## Reporting Bugs

If you find a bug, please open an issue and provide the following:

-   A clear and descriptive title.
-   A detailed description of the problem, including steps to reproduce it.
-   The expected behavior and what actually happened.
-   Any relevant logs or stack traces.
